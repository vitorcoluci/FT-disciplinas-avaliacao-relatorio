Análise discursiva com IA das questões q25 e q26

Objetivo

Este material analisa as questões discursivas da base de avaliações:

  • q25: principais aspectos positivos da disciplina;
  • q26: sugestões para melhoria da disciplina.

A análise é feita com apoio de um modelo de IA local, gerando resumos automáticos para os seguintes recortes:

  • FT como um todo;
  • por curso analítico;
  • por ano analítico;
  • por curso analítico e ano analítico.

O objetivo é transformar os comentários livres em sínteses interpretáveis, mais úteis do que nuvens de palavras isoladas.

Observação importante

Esta versão usa llama-cli local, sem Ollama.

Ela foi preparada para ser robusta contra falhas: - salva progresso parcial em disco; - salva cada lote separadamente; - salva cada grupo separadamente; - se o kernel morrer no meio, a próxima execução reaproveita o que já foi salvo.

Além disso, esta versão tenta reaproveitar a cache independentemente da máquina que a gerou. O reaproveitamento é guiado pelo recorte e pelo conteúdo dos comentários, e não pelo caminho local do modelo.

Você pode escolher o comportamento da execução com a flag COMPLETAR_CACHE_FALTANTE: - True: tenta completar localmente os grupos ainda faltantes; - False: usa apenas o que já estiver em cache; se algo estiver faltando, o relatório avisa no HTML que o grupo não foi processado.

Antes de executar no modo de completude (True), confirme que:

llama-cli --n-gpu-layers 0 --reasoning off -st \
  -m ~/modelos/gguf/deepseek-r1-distill-qwen-7b-q4_k_s.gguf \
  -p "Diga apenas: funcionando."

Conclusões metodológicas

  • Este documento troca contagem simples de palavras por resumo automático com IA.
  • A síntese é feita separadamente para q25 e q26.
  • Os agrupamentos seguem a mesma lógica do restante do projeto:
    • curso analítico;
    • ano analítico.
  • O procedimento usa resumo hierárquico:
    • primeiro resume lotes de comentários;
    • depois consolida os resumos parciais.
  • Esta versão salva progresso incremental em disco:
    • chunk_001.json, chunk_002.json, etc.;
    • meta.json;
    • final.json;
    • final.txt;
    • status.json.
  • O reaproveitamento da cache foi tornado portátil entre máquinas:
    • primeiro tenta localizar uma pasta de cache já existente para o recorte;
    • depois reaproveita final.json quando disponível;
    • a máquina que gerou a cache não precisa ser a mesma da máquina que está executando o relatório.
  • A flag COMPLETAR_CACHE_FALTANTE controla se a execução:
    • completa os grupos faltantes com IA local; ou
    • apenas usa a cache já existente e sinaliza no HTML o que ficou pendente.
